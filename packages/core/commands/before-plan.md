Write an implementation plan in HTML, but lead with the decisions I'm most likely
to tweak: data model changes, new type interfaces, and anything user-facing. Bury
the mechanical work at the bottom, I trust you on that part.

Then, while implementing, keep an implementation-notes.md file. If you hit an edge
case that forces you to deviate from the plan, pick the conservative option, log it
under "Deviations", and keep going.
